# Glossary

| Term                             | Definition                                                                                                                                                                                          |
| -------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Agent**                        | A named AI assistant with its own backend endpoint, system prompt, tool set, starter suggestions, and optional data panel. Participants add agents by registering them in `lib/agents.ts`.          |
| **Agent definition**             | The `AgentDefinition` object in `lib/agents.ts` that describes an agent to the frontend (id, name, endpoint, data panel, etc.).                                                                     |
| **Data panel**                   | The right-side pane in the UI that displays structured data emitted by agent tools. Each agent may define its own panel; if it doesn't, the layout collapses to a single chat column.               |
| **Data part**                    | A typed structured payload streamed from an agent tool alongside the text response. Shapes are defined in `WorkshopDataParts` in `lib/chat-types.ts`.                                               |
| **`WorkshopDataParts`**          | The TypeScript type map that ties data part type strings (e.g., `'data-sql-result'`) to their data shapes. The single source of truth for the data contract between backend and frontend.           |
| **`AgentDataPanelDefinition`**   | The typed object that describes a data panel: how to create initial state, how to fold incoming data parts into state, and how to render the result. Created with `defineAgentDataPanel<TState>()`. |
| **`useChatSession`**             | The React hook (`hooks/use-chat-session.ts`) that owns agent selection, `useChat` integration, panel state accumulation, and the `onData` callback.                                                 |
| **`useChat`**                    | The Vercel AI SDK hook (`@ai-sdk/react`) that manages the chat lifecycle: sending messages, receiving streamed responses, and surfacing message state.                                              |
| **Vercel AI streaming protocol** | The SSE-based wire format used to stream text deltas, tool calls, and data parts from backend to frontend. Both backend patterns (TS and Python) produce this format.                               |
| **`VercelAIAdapter`**            | The Pydantic AI utility class that wraps a `pydantic_ai.Agent` and converts its output into a Vercel AI-compatible SSE stream.                                                                      |
| **`ToolLoopAgent`**              | The AI SDK class (`ai` package) that runs an agentic tool loop — repeatedly calling tools until a stop condition is met — and streams results.                                                      |
| **`DataChunk`**                  | The Pydantic AI type used to attach structured data to a tool's return value so it is streamed as a data part on the frontend.                                                                      |
| **Starter suggestions**          | Pre-written prompt suggestions shown to the user before they type. Defined per agent in `AgentDefinition.starterSuggestions`.                                                                       |
| **Chinook database**             | A sample SQLite database of a digital music store used by the SQL Analyst agent. Downloaded via `npm run download:chinook`.                                                                         |
| **Panel state**                  | The in-memory state object managed by `useChatSession` for the active agent's data panel. Persisted for the session duration but reset on page reload.                                              |
| **`applyDataPart`**              | The reducer function in an `AgentDataPanelDefinition` that takes the current panel state and a new data part and returns the next state — analogous to a Redux reducer.                             |
